## 1. Вход в панель управления

**Адрес:** `https://ВАШ_ДОМЕН/wp-admin/`

Введите логин и пароль. После входа откроется **Панель управления** WordPress.

Если забыли пароль, нажмите **Забыли пароль?** на странице входа и укажите рабочий email.

## 3. Редактор сайта

**Путь:** **Внешний вид -> Редактор**

Редактор сайта используется для блоковой темы: здесь редактируются шаблоны, главная страница, шапка и подвал.

### Что есть в редакторе

```
Редактор сайта
├── Шаблоны
│   ├── Главная страница (Front Page)
│   ├── Страница (Page)
│   ├── Запись (Single)
│   ├── Архив (Archive)
│   └── Index / fallback
└── Части шаблона
    ├── Header
    └── Footer
```

### Как открыть нужный шаблон

1. Откройте **Внешний вид -> Редактор**.
2. Выберите **Шаблоны** или **Части шаблона**.
3. Нажмите на нужный шаблон.
4. Кликните по блоку на странице.
5. Измените поля в правой панели.
6. Нажмите **Сохранить** в правом верхнем углу.

Изменения в редакторе сайта видны только после нажатия **Сохранить**.

### Важное про сохранённые шаблоны

WordPress может сохранить изменённый шаблон в базе данных. Такой шаблон перекрывает файл темы. Если разработчик обновил файл `templates/front-page.html`, но сайт всё ещё показывает старую версию, значит в WordPress может быть сохранён override шаблона.

Для редактора это означает простое правило: если вы меняли шаблон через редактор сайта, ваши изменения имеют приоритет над файлом темы.

## 4. Шапка сайта

**Путь:** **Внешний вид -> Редактор -> Части шаблона -> Header**

Header сейчас состоит из двух блоков:

1. **Возрастное предупреждение** (`Age Disclaimer`) — верхняя полоса `18+`.
2. **Шапка сайта** (`Site Header`) — логотип и навигация.

### Age Disclaimer

| Поле | Что редактирует |
|------|------------------|
| **Текст** | Основной текст предупреждения |
| **Текст ссылки** | Текст рядом с предупреждением |
| **URL ссылки** | Ссылка, куда ведёт текст |

### Site Header

В блоке **Шапка сайта** доступны:

| Поле | Что редактирует |
|------|------------------|
| **Тёмный логотип** | Логотип для обычной/тёмной шапки |
| **Светлый логотип** | Альтернативный логотип |
| **Alt-текст** | Описание логотипа для доступности и SEO |
| **Навигационное меню** | Пункты меню, подпункты и вложенные подпункты |

Логотипы выбираются через **Медиафайлы**. Ручной ввод пути больше не нужен для обычной работы.

### Поля пункта меню

| Поле | Что означает |
|------|--------------|
| **Метка** | Текст пункта меню |
| **URL** | Ссылка пункта |
| **Иконка** | Путь к иконке, если нужна |
| **Текст бейджа** | Маленькая надпись рядом с пунктом |
| **Фон бейджа** | Цвет фона бейджа |
| **Цвет текста бейджа** | Цвет текста бейджа |
| **Бейдж "новое"** | Показывает специальную метку нового пункта |
| **Мегаменю** | Включает расширенный вид меню |
| **Второй уровень на мобильных** | Показывает пункт во втором уровне мобильного меню |
| **Ссылка футера** | Используется у вложенных пунктов как нижняя ссылка группы |

У каждого пункта верхнего уровня могут быть **Children**: подпункты и подпункты третьего уровня.

Если меню в блоке пустое, тема может использовать резервную навигацию из файлов `parts/header-sidebar-nav.html` и `parts/header-primary-nav.html`. В обычной редакторской работе лучше править меню через блок **Шапка сайта**.

## 5. Подвал сайта

**Путь:** **Внешний вид -> Редактор -> Части шаблона -> Footer**

Кликните на блок **Футер сайта**. В правой панели доступны:

| Поле | Что редактирует |
|------|------------------|
| **Логотип** | Логотип в подвале |
| **Alt-текст** | Описание логотипа |
| **Текст дисклеймера** | Текст об ответственной игре |
| **Ссылки "Информация"** | Первая колонка ссылок |
| **Ссылки "Подробнее"** | Вторая колонка ссылок |
| **Внешние ссылки** | Ссылки на внешние сайты или рынки, можно добавить флаг |
| **Навигационные ссылки футера** | Нижние ссылки подвала |
| **Копирайт** | Строка копирайта |

Каждая ссылка содержит минимум:

| Поле | Что вводить |
|------|-------------|
| **Метка** | Текст ссылки |
| **URL** | Адрес ссылки |
| **Изображение флага** | Только для внешних ссылок |

## 6. Главная страница

**Путь:** **Внешний вид -> Редактор -> Шаблоны -> Главная страница**

Текущая главная страница собирается из блоков и использует общие части шаблона **Header** и **Footer**.

### Структура главной

```
Header template part
└── Age Disclaimer
└── Site Header

Page Main
├── Hero
├── Offer List
├── Two Column Text
├── About List
├── Related Links
├── Live Odds
├── Data Table
├── Info Grid
└── News Slider

Footer template part
└── Site Footer
```

### Главное изменение по офферам

Блок **Offer List** на главной сейчас настроен как `source = query`. Это значит, что карточки казино берутся из раздела **Казино-офферы**, а не редактируются вручную внутри блока.

Для изменения названия, логотипа, бонуса, рейтинга или ссылок казино используйте раздел **Казино-офферы** в левом меню WordPress.

## 7. Офферы казино

**Путь:** левое меню -> **Казино-офферы** / **Casino Offers**

Это централизованный список казино. Блок **Offer List** может автоматически брать отсюда данные.

### Как добавить новый оффер

1. Откройте **Казино-офферы**.
2. Нажмите **Добавить новый**.
3. В поле заголовка введите название казино.
4. Заполните блок **Детали казино-оффера**.
5. В правой панели задайте **Порядок**: меньшее число показывается выше.
6. Нажмите **Опубликовать**.

### Поля оффера

| Поле | Что вводить |
|------|-------------|
| **Логотип** | Выберите изображение из Медиафайлов |
| **Рейтинг** | Число от 0 до 5, например `4.8` |
| **Бонус** | Текст бонуса |
| **Особенности** | Преимущества, по одному пункту в строке |
| **Иконки платежей** | Несколько изображений через Медиафайлы |
| **URL оффера** | Ссылка для CTA-кнопки |
| **URL обзора** | Ссылка на обзор казино |
| **Статус лицензии** | Текст лицензии или статуса |
| **Легальный оффер** | Включите чекбокс, если оффер легальный |

### Сортировка офферов

Если блок **Offer List** настроен на:

| Настройка | Как работает |
|-----------|--------------|
| `Order By = menu_order` | Сортирует по полю **Порядок** |
| `Order = ASC` | Меньшие числа идут выше |
| `Limit = 15` | Показывает максимум 15 офферов |

### Как скрыть оффер

1. Откройте нужный оффер.
2. В правой панели измените статус на **Черновик**.
3. Нажмите **Обновить**.

Черновики не показываются в автоматическом списке.

## 8. Статьи и новости

**Путь:** левое меню -> **Записи**

### Как опубликовать статью

1. Нажмите **Добавить новую запись**.
2. Введите заголовок.
3. Добавьте текст и блоки статьи.
4. В правой панели заполните:
   - **Изображение записи** — картинка для карточек и слайдера;
   - **Рубрики** — категория статьи;
   - **Метки** — теги, если нужны.
5. Нажмите **Опубликовать**.

### News Slider

Блок **News Slider** может работать в двух режимах:

| Режим | Что происходит |
|-------|----------------|
| **query** | Автоматически показывает опубликованные записи |
| **manual** | Показывает вручную заданные элементы |

Если выбран `query`, новые опубликованные статьи появятся автоматически, если подходят под настройки блока. Если выбран `manual`, элементы нужно добавлять в самом блоке.

## 9. Справочник блоков

Все кастомные блоки находятся в категории **Новая тема**.

### Age Disclaimer

Полоса с предупреждением в самом верху сайта.

| Поле | Описание |
|------|----------|
| **Текст** | Основное предупреждение |
| **Текст ссылки** | Текст ссылки |
| **URL ссылки** | Адрес ссылки |

### Page Main

Основная обёртка контента страницы. Обычно редактору не нужно добавлять её вручную.

| Поле | Описание |
|------|----------|
| **JSON-LD schema** | Schema-разметка страницы |

Меняйте schema только если понимаете, зачем это нужно. Ошибка в JSON может сломать разметку.

### Hero

Главный баннер страницы с H1.

| Поле | Описание |
|------|----------|
| **Заголовок** | H1 страницы |
| **Текст** | Описание под заголовком |
| **Фоновое изображение** | Выбирается через Медиафайлы |
| **Вариант** | `Темный` или `Светлый` |
| **Цвет оверлея** | HEX-цвет, например `#000000` |
| **Прозрачность оверлея** | От 0 до 90 |
| **Верхний отступ** | Высота отступа над текстом в пикселях |

На каждой важной странице должен быть понятный H1.

### Offer List

Список казино-офферов.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок секции |
| **Intro** | Вступительный текст |
| **Источник** | `query` или `attributes` |
| **Limit** | Максимальное число карточек |
| **Order By** | `menu_order`, `date`, `title`, `rating`, `bonus` |
| **Order** | `ASC` или `DESC` |
| **Show Legal Label** | Показывать юридическую метку |

Рекомендуемый режим для главной: `source = query`. Тогда карточки управляются через **Казино-офферы**.

Режим `attributes` нужен для ручного списка прямо внутри блока. Используйте его только если список должен отличаться от общего справочника офферов.

### Offer Card

Одна карточка казино. Используется внутри ручного Offer List или отдельно.

| Поле | Описание |
|------|----------|
| **Name** | Название казино |
| **Logo** | Логотип через Медиафайлы |
| **Rating** | Рейтинг от 0 до 5 |
| **Bonus** | Текст бонуса |
| **Features** | Преимущества, по одному в строке |
| **Payments** | Иконки способов оплаты |
| **Offer URL** | Ссылка основной кнопки |
| **Review URL** | Ссылка на обзор |

Для главной страницы обычно не нужно редактировать Offer Card вручную.

### Two Column Text

Текстовая SEO-секция в две колонки.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок секции |
| **Background** | Цвет фона |
| **Items** | Группы текста |

Каждый item содержит:

| Поле | Описание |
|------|----------|
| **Heading** | Подзаголовок |
| **Left** | Текст левой колонки |
| **Right** | Текст правой колонки |

### About List

Список преимуществ или критериев с иконками.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок секции |
| **Background** | Цвет фона |
| **Items** | Пункты списка |

Каждый item содержит **Title** и **Description**.

### Info Grid

Сетка информационных блоков.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок секции |
| **Background** | Цвет фона |
| **Items** | Ячейки сетки |

Каждый item содержит:

| Поле | Описание |
|------|----------|
| **Text** | Основной текст |
| **Link Label** | Текст ссылки |
| **Link URL** | URL ссылки |

Если **Link Label** пустой, ссылка не показывается.

### Content Section

Универсальная секция для текста, about-формата или info-формата.

| Поле | Описание |
|------|----------|
| **Kind** | Тип секции: text, about или info |
| **Title** | Заголовок |
| **Text** | Основной текст |
| **Image** | Изображение через Медиафайлы |
| **Background** | Цвет фона |
| **Items** | Повторяемые элементы, если используются |

На текущей главной чаще используются отдельные блоки **Two Column Text**, **About List** и **Info Grid**.

### Related Links

Секция ссылок на другие страницы.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок |
| **Intro Text** | Вступительный текст |
| **Variant** | `cards`, `grid` или `footer` |
| **Background** | Цвет фона |
| **Items** | Список ссылок |

Каждый item содержит:

| Поле | Описание |
|------|----------|
| **Image** | Картинка или иконка через Медиафайлы |
| **Title** | Заголовок карточки |
| **Text** | Описание |
| **URL** | Ссылка |
| **Link Label** | Текст кнопки |

### Data Table

Адаптивная таблица.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок таблицы |
| **Background** | Цвет фона |
| **Mobile behavior** | `scroll` или `stacked` |
| **Заголовки столбцов** | Колонки через символ `|` |
| **Строки** | Каждая строка с новой строки, ячейки через `|` |
| **Выделение строки** | Кнопка со звездой рядом со строкой |
| **Выделение ячейки** | Звезда в ячейке |

Пример:

```text
Kazino | Licencija | Mokejimai
Betano | Patikrinta | Korteles, el. pinigines
Solverde | Patikrinta | Korteles, PayPal
```

### News Slider

Слайдер новостей или ручных карточек.

| Поле | Описание |
|------|----------|
| **Заголовок блока** | Заголовок секции |
| **Контент source** | `query` или `manual` |
| **Включить слайдер** | Включает горизонтальный слайдер |

Если выбран `query`:

| Поле | Описание |
|------|----------|
| **Тип записи** | Обычно `post` |
| **Slug категории** | Фильтр по категории, можно оставить пустым |
| **Количество записей** | От 1 до 20 |

Если выбран `manual`, каждый элемент содержит изображение, category, дату, заголовок и URL.

### Live Odds

Виджет спортивных прогнозов.

| Поле | Описание |
|------|----------|
| **Заголовок секции** | Заголовок блока |
| **Включить слайдер** | Включает слайдер прогнозов |
| **Путь к логотипу провайдера** | Относительный путь темы или абсолютный URL |
| **Alt логотипа провайдера** | Описание логотипа |
| **URL CTA по умолчанию** | Ссылка для всех прогнозов, если у прогноза нет своей |
| **Текст кнопки** | Текст общей CTA-кнопки секции |
| **URL кнопки** | Ссылка общей CTA-кнопки |
| **Прогнозы** | Список матчей |

Каждый прогноз содержит:

| Поле | Описание |
|------|----------|
| **Команда A** | Первая команда |
| **Команда B** | Вторая команда |
| **Лига** | Турнир или лига |
| **Дата события (ISO UTC)** | Например `2026-06-10T19:00:00Z` |
| **Маркет** | Рынок ставки |
| **Коэффициент** | Например `1.82` |
| **URL CTA** | Индивидуальная ссылка, если нужна |
| **Live-обновления** | Отметка для live-матча |

Даты вводятся в UTC. Например, `2026-06-10T19:00:00Z` означает 19:00 UTC.

### FAQ и FAQ Item

**FAQ** — контейнер секции вопросов.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок секции |

**FAQ Item** — один вопрос и ответ.

| Поле | Описание |
|------|----------|
| **Question** | Вопрос |
| **Answer** | Ответ |

### Card Grid и Link Card

**Card Grid** — контейнер карточек.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок сетки |

**Link Card** — одна карточка-ссылка.

| Поле | Описание |
|------|----------|
| **Title** | Заголовок |
| **Text** | Описание |
| **Image** | Изображение через Медиафайлы |
| **URL** | Ссылка |

## 10. Частые задачи

### Изменить бонус у казино

1. Откройте **Казино-офферы**.
2. Найдите нужное казино.
3. Откройте его на редактирование.
4. В блоке **Детали казино-оффера** измените **Бонус**.
5. Нажмите **Обновить**.

Если главная использует `Offer List -> source = query`, изменение появится в списке автоматически.

## 11. Полезные правила

### Картинки и логотипы

В новых полях темы изображения выбираются через **Медиафайлы**. Это касается:

- логотипов header/footer;
- Hero background;
- логотипов офферов;
- иконок платежей;
- изображений в Related Links, News Slider, Link Card и Content Section;
- флагов во внешних ссылках футера.

Старые относительные пути вида `assets/img/...` всё ещё поддерживаются для импортированного контента, но редакторам лучше использовать Медиафайлы.

### Ссылки

| Тип ссылки | Формат |
|------------|--------|
| Внутренняя страница | `/betano-portugal/` |
| Якорь или пустой родительский пункт меню | `#` |
| Внешний сайт | `https://example.com/` |
| Аффилиат-ссылка | Полный URL с `https://` |

### HTML в текстовых полях

В полях, которые поддерживают HTML, можно использовать:

```html
<strong>жирный текст</strong>
<a href="/page/">ссылка</a>
<br>
<span class="text--main-color">зелёный текст</span>
```

Не вставляйте скрипты, iframe и сложную вёрстку в текстовые поля.

### Цвета

| Цвет | Код | Где используется |
|------|-----|------------------|
| Белый / пусто | пустое поле | Обычный фон |
| Светло-серый | `#f2f2f3` | Чередование секций |
| Зелёный акцент | `#15a96a` | Акцентный текст и ссылки |

### Когда изменения становятся видны

| Где меняли | Когда видно |
|------------|-------------|
| **Казино-офферы** | После **Обновить** |
| **Записи** | После **Опубликовать** или **Обновить** |
| **Редактор сайта** | После **Сохранить** |
| **Медиафайлы** | Сразу после выбора и сохранения блока/оффера |

Если сайт долго показывает старую версию, возможно сработал кэш. Сообщите разработчику, что нужно очистить WordPress object cache или nginx cache.
